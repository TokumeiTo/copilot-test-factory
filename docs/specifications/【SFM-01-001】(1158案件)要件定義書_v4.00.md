# システムの目的と目標

顧客の利便性向上のため、非課税口座の簡易開設時に併せて先日付のNISA注文（買付、応募、定時定額）を受付可能とするよう対応する。

# 業務要件

## 現行業務・システムの問題点と改善点

現状は、非課税口座の簡易開設申込日の翌日からNISA注文が受付可能となっている。そのため、NISA注文を受付するには翌日以降に再度顧客とのアポイントを取る必要があった。今回の対応で、非課税口座の簡易開設申込時に先日付のNISA注文が受付可能となるようシステム改修を行うことで、顧客の利便性向上を図る。

制度要件はないため、記載事項なし。

# 制約・前提条件

## システム機能に関する前提条件

前提事項は、以下4点とする。

1. 非課税口座の簡易開設時のみ先日付NISA注文を受付可能とする。（再開設または金融機関変更による非課税口座開設は対象外。）
2. 買付・応募・定時定額機能において、当日のNISA注文は従来通り不可であること。
3. タブレットで非課税口座開設機能を提供している9社（大東銀行、愛媛銀行、福島銀行、荘内銀行、香川銀行、組連銀行、北都銀行、三十三銀行、DIR銀行）を対象とする。
4. タブレットで非課税口座開設を受付した当日に、非課税口座管理サブシステム（LISA）へ非課税口座開設データを登録する事務運用であること。

## テストに関する前提条件

# システムの概要

## 事務フロー／業務フロー

非課税口座の簡易開設申込日に先日付のNISA注文を受付可能とした場合の事務フローは、以下のとおり。

従来の業務フローから変更している箇所は、太字で記載する。

①タブレットにて非課税口座の簡易開設を行う。

**②先日付のNISA注文（買付、応募、定時定額）をタブレットまたはFRから行う。**

**③上記①で受付した非課税口座開設に誤りがある場合は、検印却下後に再度受付をするかLISAシステムへ登録する際に修正を行う。非課税口座開設の検印を却下する場合は、紐づく先日付NISA注文の検印却下または取消を行う。**

④上記①で受付した非課税口座開設について、検印を行う。

⑤軽技より、非課税口座管理サブシステム登録用データを抽出する。

⑥非課税口座管理サブシステム（LISA）へ非課税口座開設申込データを登録する。

**⑦翌営業日にFRの進捗状況一覧画面より、先日付NISA注文のエラー有無を確認する。**
