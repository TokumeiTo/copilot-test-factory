# 本ドキュメントの目的
【本ドキュメントの目的】
本ドキュメントは品質向上を目的とした資料である。概要設計を共通フォーマットにまとめることで見積り、設計時に
観点の漏れなどが起きないようにする。また、「なぜ」「どうして」そのような設計方針としたのかを整理し、
明記することで検討時のプロセスに誤りがないかをレビューできるようにする。
（※）要件定義プロセスと見積プロセスが逆転することが多々あるが、その場合は設計方針書にて要件整理を行う。
 　　 見積プロセス前に要件定義書を作成している場合、設計方針書で要件定義書と重複する内容については
　　「要件定義書を参照」で可とするが、設計方針書には主になぜこのような設計としたのか、検討したプロセスを
　　 しっかりと明記することが目的であることを意識すること。

# 本ドキュメントの目的
【本ドキュメントの対象者】
開発チーム（プロパ、ニアショア、オフショア）向け。

【本ドキュメントのルール】
受託案件は見積り時、保守案件はプロジェクト計画時、本ドキュメントを作成する。
作成した本ドキュメントは受託案件の場合は見積りフォルダ、保守案件の場合は要件定義フォルダに配置する。
受託案件の場合、プロジェクト始動後本ドキュメントを要件定義フォルダ配下に配置する。

本ドキュメントは必ず課長レビューを行う。

各ページに対して対応要否の確認を行い、要件がないシートは「記載事項なし」と記載する。
設計着手後決まる場合はその旨を記載し、設計着手後更新するものとする。
（設計の方針を記すドキュメントであるため、基本設計が完了するまでに本ドキュメントも必ず更新する）
また、プロジェクト始動後、要件の変更があった場合についても本ドキュメントを更新する。

プロジェクトメンバーは設計着手前に本ドキュメントを確認し、要件、設計方針を理解する。

# １．案件概要
【目的】
・非課税口座開設同日のNISA先日付注文を可能とする

【背景】
・現状、非課税口座開設とNISA注文は、別々の日に申し込む必要がある。
　これは、MFより非課税口座開設登録結果を夜間に受領し、NISA注文に
　必要なデータがSONAR-FR/FFに登録されることで、翌日以降に注文が
　可能となるためである。
　顧客の利便性を向上させるため、非課税口座開設同日にNISA注文が行える
　ようシステム改修を行う。
